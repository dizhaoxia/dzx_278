## 1. 产品概述

一个基于 WebRTC 的屏幕共享演示工具，用于在浏览器中演示端到端的 P2P 实时画面传输。A 端通过 `getDisplayMedia` 捕获屏幕并推流，B 端通过 `<video>` 标签实时接收并播放画面。信令交换由 Node.js + Express + `ws` 搭建的 WebSocket 信令服务器承担，负责房间创建与加入、SDP 协商与 ICE 候选交换。

- 核心目标：验证 WebRTC 原生 `RTCPeerConnection` 在浏览器间建立 P2P 连接、传输屏幕画面的完整链路
- 目标用户：WebRTC / 实时音视频方向的学习者与原型开发者
- 价值：提供一个零外部依赖（无 STUN/TURN 之外的托管服务）、可读、可复现的 WebRTC 屏幕共享参考实现

## 2. 核心功能

### 2.1 用户角色

| 角色 | 接入方式 | 核心权限 |
|------|----------|----------|
| A 端（主播 / 推流端） | 创建房间，获得房间号 | 捕获屏幕、发起 Offer、推送屏幕画面 |
| B 端（观众 / 拉流端） | 输入房间号加入房间 | 接收画面、应答 Answer、播放远端画面 |

### 2.2 功能模块

1. **信令控制台（首页 / 房间入口）**：角色选择、房间创建、房间号输入与加入、连接状态实时展示
2. **A 端屏幕共享工作台**：屏幕捕获按钮、本地预览、WebRTC 连接状态、编码格式与统计信息
3. **B 端播放工作台**：房间号、远端画面播放区、连接状态、码率 / 分辨率统计

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 信令控制台 | 角色切换 | 在 A 端 / B 端之间切换，决定后续操作面板 |
| 信令控制台 | 房间操作 | A 端创建房间并生成房间号；B 端输入房间号加入 |
| A 端工作台 | 屏幕捕获 | 调用 `getDisplayMedia` 获取屏幕流并本地预览 |
| A 端工作台 | 推流控制 | 建立 `RTCPeerConnection`，设置 VP8/H.264 编码偏好，创建并发送 Offer |
| A 端工作台 | 状态面板 | 展示 ICE/连接状态、编码格式、发送码率与分辨率 |
| B 端工作台 | 接收控制 | 加入房间后建立 `RTCPeerConnection`，接收 Offer 并回送 Answer |
| B 端工作台 | 画面播放 | 通过 `<video>` 自动播放远端屏幕画面 |
| B 端工作台 | 状态面板 | 展示连接状态、接收码率、分辨率与候选类型 |

## 3. 核心流程

**A 端推流流程**：A 端创建房间 → 信令服务器分配房间号并通知 A 端 → A 端点击"捕获屏幕"调用 `getDisplayMedia` → A 端将屏幕流加入 `RTCPeerConnection` → 创建 Offer → 通过信令服务器转发至 B 端 → 交换 ICE 候选 → 连接建立后开始 P2P 传输屏幕画面。

**B 端拉流流程**：B 端输入房间号加入 → 信令服务器通知 A 端有 B 端加入 → B 端创建 `RTCPeerConnection` 并监听 `ontrack` → 接收 A 端 Offer → 创建 Answer 并回传 → 交换 ICE 候选 → `ontrack` 触发后画面绑定到 `<video>` 播放。

## 4. 用户界面设计

### 4.1 设计风格

- **设计方向**：信号实验室 / 广播控制台（Signal Lab）风格 —— 深色仪表盘、等宽显示字体、霓虹信号色，营造"实时链路监控台"的氛围，契合实时音视频技术语境
- **主色与配色**：近黑底色（#07090b / #11161a）+ 电光薄荷绿信号色（#00ff9c）作为唯一强调色；告警与状态色使用琥珀（#ffb020）与品红（#ff3366）
- **按钮风格**：低圆角矩形（4px）、细边框、hover 时发出微弱信号光晕；主操作按钮使用信号绿描边 + 半透明填充
- **字体**：标题使用 `Major Mono Display`（极具技术感的等宽显示字体）；正文使用 `IBM Plex Sans`；标签 / 码值 / 状态使用 `JetBrains Mono`
- **布局风格**：基于网格的仪表盘式布局，左控制栏 + 右主显示区；视频区使用扫描线 / 角标十字准星等"取景器"装饰
- **图标风格**：细线条线性图标，配合等宽状态标签与闪烁信号灯动效

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|----------|
| 信令控制台 | 顶部标识 | 信号灯动效、项目标题 Major Mono Display、端口与连接状态徽章 |
| 信令控制台 | 角色切换 | A 端 / B 端 两个大号切换卡，选中态信号绿描边 + 光晕 |
| 信令控制台 | 房间操作 | 房间号输入框（等宽字体）、创建 / 加入按钮、房间号生成动画 |
| A 端工作台 | 本地预览 | 视频预览框 + 取景器角标 + 屏幕捕获按钮 |
| A 端工作台 | 状态仪表盘 | 连接状态信号灯、ICE 状态、编码格式、发送码率 / 分辨率读数 |
| B 端工作台 | 远端播放 | 大幅 `<video>` 播放区 + 连接前占位扫描动画 |
| B 端工作台 | 状态仪表盘 | 连接状态、候选类型、接收码率 / 分辨率读数 |

### 4.3 响应式

- 桌面优先设计：主工作台在宽屏下采用"左控制 + 右显示"双栏布局
- 窄屏自适应：≤ 960px 时折叠为单栏，控制面板上、视频区下
- 触屏优化：按钮与输入框加大点击区域，避免悬停依赖的交互
